Scope: this is a prompt-alignment-checked regression-gated Page Connector V58 build: V8 kernel first, with V7/prompt intentions re-added as compact modules. It has localdata vault, identity, manual WebRTC first contact, negotiated control/binary channels, binary file pump with backpressure, route-vault recording, bounded eval, SignalMux hooks, SyncPlanner, and MediaManager. It is still not the full production Auto-Rendezvous/Sync release, but visible workflows are treated as no-regression UX requirements.

    Clipboard bridge / paste pipe
    
Main workflow: Device A copies something locally → clicks Send my clipboard → peer clipboard. If Device B keeps auto-apply remote clipboard on (default), Device B's system clipboard becomes the same thing, so pasting on B pastes what A copied. Reverse direction uses Get peer clipboard → this clipboard. Browser permission may still require a click/prompt; blocked writes stay visible in the received buffer. Rich file/image clipboard-to-clipboard stays in memory first so it can be offered to navigator.clipboard.write before folder-save fallback.

6. Connection / SignalMux / relay order
  
V8-native extraction of V7 link/socket/Nostr/MQTT idea: relay order and connection-method details live here; one method list feeds a SignalMux. This slice packs/unpacks relay payloads encrypted by the editable encrypt string + room context and keeps method order; custom WSS/manual packet transport are wired with optional auto-apply; Nostr/MQTT/custom-WSS/manual are wired; link/note relay is also available as a V7-style compatibility fallback.

8. MediaManager / voice video screen hooks
  
Media is V8-native: addTrack/removeTrack uses the perfect-negotiation path instead of ad-hoc SDP code. Bulk files stay on bin channel; media rides WebRTC RTP and shares network capacity. This section also handles crash/stop-share cases: ended/mute/unmute events clean up senders and trigger renegotiation instead of leaving dead tracks.
