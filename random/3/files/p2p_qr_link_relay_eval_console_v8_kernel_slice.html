Scope: this is the first organized v8 kernel slice: localdata vault, device identity, manual WebRTC first contact, negotiated control/binary channels, binary file pump with backpressure, route-vault recording, bounded eval, and reconnect scheduler hooks. It is not yet the full no-library Nostr/MQTT Auto Reconnect implementation.

2. Auto Start Connect / manual signaling fallback
Both devices create the same negotiated channels. Device A makes offer; Device B applies offer and makes answer; Device A applies answer. Future v8 Auto Start/Auto Reconnect should put these same SDP boxes through hidden encrypted relays.
2b. CryptoBox for public/hidden relay payloads
This is the v8 signaling encryption core: derive key once per room/randomizer/password, use fresh IV per message, include msgId/time for replay filtering. It packs/unpacks the manual SDP code so future Nostr/MQTT/WSS relays can carry ciphertext only.
3. Auto Reconnect / route vault
This is the v8 reconnect direction implemented as hooks: identity vault, route-vault candidate capture, restartIce trigger, and 1/sec scheduler. Hidden relay rooms are logged/derived but not yet transported by built-in Nostr/MQTT.
