Must-fix findings:
1. Public rendezvous priority list is invisible at page load.
   Cause: renderPublicMethods() exists but is not called in the init block.
   Impact: violates priority checkbox / reorder UI requirement even though internal methods still exist.
   Fix: call renderPublicMethods(); updatePublicRoomPreview(); near init.

2. “Each input box has file mode” is only partially implemented.
   Fields with 🔗: localCode, remoteCode, relayUrl, publicRoomSeed, publicRoomSalt, socketUrl, socketRoomSeed, socketRoomSalt, customText, evalCode.
   Fields without 🔗: signalCryptoKey, signalCryptoRounds, publicMethodSeconds, publicRetryMaxSeconds, richSendFilter, richSourcePathHint, richReceiverSubfolder, richReceiverName, richReceiverPathHint, richBackupRootHint.
   Fix depends on literalness: either add 🔗 to all text inputs or define exceptions for settings/filter fields.

3. Strict “no library unless QR clicked” is not fully obeyed if Auto Connect is used.
   QR library is lazy-loaded only by Make QR, but Auto Connect can import/load Nostr tools, MQTT.js, and PeerJS libraries when those public rendezvous methods are attempted.
   If the rule means “no page-load libraries,” this is OK. If it means “no external library at all except QR button,” Auto Connect violates it.

4. Random-device/random-internet is not guaranteed.
   Nostr/MQTT/PeerJS help signaling/discovery, but the normal WebRTC DataChannel still uses STUN only. Without TURN, some NAT/firewall/mobile-carrier cases can fail.

5. Rich transfer is useful but not raw-binary optimized.
   Built-in file send uses JSON + base64 chunks. This is safe below the discovered max-message-size cliff, but slower/larger than raw ArrayBuffer chunking. Raw binary messages are currently logged/ignored by handleRichBinaryPayload().

6. Voice/video/screen share only works on normal RTCPeerConnection path.
   PeerJS fallback is bound as an app channel but does not create pc/media negotiation support. The UI logs that limitation.

7. Two collapse checkboxes jointly gate auto-collapse.
   Both autoCollapseOnConnect and publicAutoCollapse must be checked. Defaults are checked, so happy path is OK, but disabling either silently disables collapse.
