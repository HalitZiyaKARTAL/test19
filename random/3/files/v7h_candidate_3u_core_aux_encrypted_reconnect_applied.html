Fast usage
Expected fast path: open this page on both devices, then press Auto Connect on both. The button becomes Stop auto; the app keeps trying enabled public WSS/socket rendezvous methods in priority order until connected or stopped, reports failures, and collapses setup UI after DataChannel opens.

Manual/QR fallback: Device A makes host code → Device B scans/pastes it → Device B makes reply → Device A scans/pastes reply → finish.

Link relay: default link is note.ms, but the field can be replaced by any compatible note/proxy URL. Enable Use link intermediary on both devices.

Socket/auto rendezvous: default room id is cross_talk_room1 and randomizer is off, so the literal room is cross_talk_room1. For private deterministic rooms, put the visible room id in the first box, the shared randomizer phrase/key in the second box, and turn randomizer on on both devices. Example: room id apple + randomize using orange → same hidden room; randomizer off → room is only apple.

Auto-collapse is on by default: after DataChannel opens, sections above Text / ping collapse to remove setup clutter.

Eval console is under Global log so received/sent text and eval output are visible in the same lower area. Remote eval only runs when Accept remote eval is checked on the receiving device.

🔗 buttons beside fields open a file picker and only populate that specific box. They do not run, send, connect, or evaluate anything automatically.

public crypto on
Applies to public signaling bodies written through link relay, socket relay, Nostr, and MQTT rendezvous. The default key encrypt hides data from casual listeners but is not secret because it is visible in the HTML; change it on both devices for real privacy. Direct link relay tries GET + POST. If read/write fails but manual page works, paste a CORS proxy URL directly into the Relay URL field. Encryption here protects the SDP/signaling content placed in public locations, but both sides must use the same key string.
